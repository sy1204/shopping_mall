제시해주신 《Do it! 웹사이트 기획 입문》의 목차는 기획의 기초부터 실무 산출물 작성까지 매우 체계적으로 구성되어 있습니다. 요청하신 대로 전체 기획 프로세스와 상세 IA(정보 구조도)를 정리해 드립니다.

---

## 1. 웹 서비스 기획 및 구현 전체 프로세스

기획 초기 단계부터 오픈 후 운영까지의 과정을 단계별 활동과 주요 산출물로 정리했습니다.

| 단계 | 주요 활동 | 주요 발생 서류 (산출물) |
| --- | --- | --- |
| **1. 전략 및 개념 정의** | 시장 조사, 벤치마킹, 서비스 컨셉 수립 | 서비스 기획 초안, 벤치마킹 보고서 |
| **2. 요구사항 정의** | 서비스 필요 기능 확정 및 정책 수립 | **기능 정의서**, **정책 정의서** |
| **3. 구조 설계 (IA)** | 서비스 메뉴 구조 및 정보 설계 | **IA (Information Architecture)** |
| **4. 상세 설계 (화면)** | 화면별 UI 구성 및 상세 로직 설계 | **화면 정의서 (Storyboard/Wireframe)** |
| **5. 일정 수립** | 개발 및 디자인 일정 산출 | **일정 산출표 (WBS)** |
| **6. 제작 및 구현** | 디자인, 퍼블리싱, 백엔드 개발 진행 | 디자인 가이드, 소스 코드 |
| **7. 검증 및 완료** | QA(품질 보증), 결함 수정 | **테스트 케이스(TC)**, 완료 보고서 |
| **8. 운영 및 유지보수** | 사이트 오픈, 데이터 분석, 운영 업무 | 운영 매뉴얼, 통계 보고서 |

## 2. 화면 정의서 기반 상세 IA (Information Architecture)

목차의 4장(이용자)과 5장(관리자) 내용을 바탕으로 Depth를 구분하여 IA 형태로 재구성했습니다.

### [사용자 페이지 (User Interface)]

| 1Depth | 2Depth | 3Depth (주요 화면 및 기능) |
| --- | --- | --- |
| **공통** | 헤더/푸터 | GNB 서브 메뉴, 푸터 정보 |
| **메인** | 메인 화면 | 메인 배너, 상품/게시판 호출, 검색 결과 |
| **회원** | 회원가입 | 약관 동의, 정보 입력, 타당성 검증, 본인 인증 |
|  | 계정 찾기 | 아이디 찾기, 비밀번호 찾기 |
|  | 로그인 | 로그인 화면 |
| **스타일 숍** | 상품 소개 | 상품 목록, 상품 상세(구매/탭메뉴/리뷰/문의/배송안내) |
| **오픈 숍** | 상품 소개 | 상품 목록, 상품 상세(신고 팝업, 댓글 기능) |
| **주문/결제** | 구매 프로세스 | 구매 목록, 주문/배송 정보 입력, PG 연결, 주문 완료 |
| **커뮤니티/이벤트** | 이벤트 | 이벤트 목록, 이벤트 상세 |
|  | 게시판 | 공지사항, 커뮤니티 목록/상세/글쓰기 |
| **고객센터** | 고객 지원 | 자주 묻는 질문, 상품 문의, 1:1 문의 작성 |
| **마이페이지** | 쇼핑 정보 | 장바구니, 주문/배송 조회(목록/상세), 교환/환불(신고/내역), 포인트 |
|  | 오픈 숍 관리 | 상품 등록, 거래 상품 목록, 주문 관리(목록/상세) |
|  | 활동 내역 | 1:1 문의 내역, 나의 리뷰, 나의 상품 문의 |
|  | 개인정보 | 비밀번호 확인/변경, 개인정보 수정, 회원 탈퇴 |

---

### [관리자 페이지 (Admin/Back-office)]

| 1Depth | 2Depth | 3Depth (주요 화면 및 기능) |
| --- | --- | --- |
| **공통/접속** | 로그인 | 관리자 로그인, GNB, 서브 레이아웃 |
| **주문 관리** | 스타일 숍 주문 | 주문 관리(목록/상세), 취소 관리, 교환 신청 관리 |
|  | 오픈 숍 주문 | 주문 현황(목록/상세), 포인트 지급 현황 |
| **게시판 관리** | 운영 관리 | 공지사항(스타일숍/고객센터), 커뮤니티, 상품 문의, 1:1 문의 |
|  | 콘텐츠 관리 | 리뷰 관리, FAQ, 이벤트, 허위 상품 접수, 게시판 구분 관리 |
| **회원 관리** | 회원 운영 | 회원 목록, 회원 상세, 탈퇴 회원 관리 |
|  | 메시징 관리 | 메일 관리, SMS 관리, 대량 메일 발송 |
| **전시 관리** | 메인/배너 | 메인 상품 관리, 배너 관리 |
| **상품 관리** | 스타일 숍 상품 | 상품 등록, 상품 목록, 카테고리 관리 |
| **통계 관리** | 데이터 분석 | 방문자, 회원수, 상품 조회수, 판매 순위, 포인트, 탈퇴 사유 |
| **시스템 관리** | 기본 정책 | 약관 관리, 정책 관리, 결제 방법 관리, 메뉴 접근 권한 |
| **사내 관리** | 사내 소통 | 사내 게시판, 관리자 메인 |

### 💡 기획자로서 참고할 점

* **사용자 IA**는 소비자의 **'구매 여정'**에 최적화되어 있고, **관리자 IA**는 운영자의 **'업무 프로세스'**에 맞추어 설계되어 있습니다.
* 화면 정의서를 작성할 때는 위 IA의 각 하위 항목(3Depth) 마다 **'어떤 데이터가 DB에서 호출되는지'**, **'사용자가 버튼을 눌렀을 때 어떤 정책이 실행되는지'**를 기술하게 됩니다.

이 IA 구조를 바탕으로 실제 화면 설계(와이어프레임)를 진행해 보시겠어요? 혹은 특정 메뉴(예: 검색 결과 화면)의 상세 정책을 어떻게 세우는지 궁금하신가요?
